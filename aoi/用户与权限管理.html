用户与权限管理

用户与权限管理

概要介绍

Qubit用户管理系统采用集中式数据管理，用户信息集中保存于SPC服务器上的用户数据库中，所有的AOI主机可以共享用户信息，避免在每台AOI主机上重复创建用户信息。

Qubit用户管理系统采用了简洁安全的权限分组，系统的超级管理员与服务器Windows操作系统的管理员绑定，从而避免了与Windows操作系统的管理员不一致而可能带来的安全隐患。

Qubit用户管理系统采用了灵活方便的配置方式，可以在AOI主机单独开启或者关闭权限控制，让无需这一功能的客户使用起来毫无负担。

如果在AOI主机开启了权限控制，操作人员在未登陆状态下，可以执行以下操作：

  1. 运行AOI主程序。
  2. 使用已有的AOI检测程序。

如果操作人员希望执行以下操作来改变现有数据，则需要"AOI工程师"授权：

  1. 保存AOI检测程序。
  2. 使用修改后的AOI检测程序。

登录SPC服务器管理用户

  1. 在浏览器中访问：http://SPC_Server/user ，其中SPC_Server为SPC服务器名称。

  2. 如果SPC服务不在本地，则需要使用SPC服务器管理员账号登录。

  3. 在用户管理页面上，点击 [ 新添用户 ] ，在"新添用户"页面，输入新用户的用户名、密码、权限分组，其中默认权限分组为"AOI工程师"，点击 [ 保存 ] 。

  4. 管理员可以在用户管理页面上编辑、删除已有用户。

在AOI主机上开启权限控制

  1. 在AOI主机上使用管理员权限运行Tools\enable-aoi-user-management.bat，开启。

  2. 权限控制开启后，操作人员在AOI主机上运行AOI程序时，可以在 [ AOI ] 菜单上看到 [ 登录 ] 、 [ 修改密码 ] 和 [ 注销 ] 菜单项处于启用状态。

  3. 权限控制开启后，AOI程序会在操作人员执行以下操作时进行权限检查：保存检测程序、运行修改后的检测程序、提交检测结果。

在AOI主机上关闭权限控制

  1. 在AOI主机上使用管理员权限运行Tools\disable-aoi-user-management.bat，关闭权限控制。

  2. 权限控制关闭后，操作人员可以看到 [ AOI ] 菜单上 [ 登录 ] 、 [ 修改密码 ] 和 [ 注销 ] 菜单项都处于禁用状态。

使用建议

由于Qubit用户管理系统默认SPC服务器上的Windows操作系统管理员为超级用户，具有对所有AOI用户的管理权限，并且AOI主机上的Windows操作系统管理员有权限开启和关闭AOI程序的权限检查，所以建议普通操作人员在AOI主机和SPC服务器上都应该使用非管理员账号登录Windows操作系统。
